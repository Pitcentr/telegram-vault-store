# Настройка переменных после установки

## Проблема

При установке Umbrel не показал форму для ввода переменных окружения, и приложение не запускается с ошибкой:
```
Missing required environment variables: TG_TOKEN, PB_PASSWORD, MASTER_PASSWORD, ALLOWED_USERS
```

## Решение

Настроить переменные можно **после установки** через настройки приложения.

## Способ 1: Через Umbrel UI (рекомендуется)

### Шаг 1: Откройте настройки приложения

```
Umbrel → Apps → Telegram Vault → Settings (⚙️ иконка)
```

### Шаг 2: Найдите раздел Environment Variables

Прокрутите вниз до раздела "Environment Variables" или "Configuration"

### Шаг 3: Добавьте переменные

Нажмите "Add Variable" или "Edit" и добавьте каждую переменную:

```
Имя: APP_TG_TOKEN
Значение: <ваш токен от @BotFather>

Имя: APP_PB_ADMIN
Значение: admin@vault.local

Имя: APP_PB_PASSWORD
Значение: <ваш пароль>

Имя: APP_MASTER_PASSWORD
Значение: <длинный мастер-пароль>

Имя: APP_ALLOWED_USERS
Значение: <ваш Telegram ID>

Имя: APP_POCKETBASE_URL
Значение: http://pocketbase_server:8090
```

### Шаг 4: Сохраните и перезапустите

1. Нажмите "Save" или "Apply"
2. Нажмите "Restart" для перезапуска приложения
3. Проверьте логи

## Способ 2: Через файл (если UI не работает)

### Шаг 1: Подключитесь к Umbrel по SSH

```bash
ssh umbrel@umbrel.local
# Или используйте IP адрес
ssh umbrel@192.168.1.xxx
```

### Шаг 2: Найдите директорию приложения

```bash
cd ~/umbrel/app-data/vault-telegram-vault
```

### Шаг 3: Создайте файл с переменными

```bash
nano .env
```

### Шаг 4: Добавьте переменные

Вставьте следующее содержимое (замените значения на свои):

```env
APP_TG_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
APP_PB_ADMIN=admin@vault.local
APP_PB_PASSWORD=YourSecurePassword123
APP_MASTER_PASSWORD=YourVeryLongMasterPassword123456789
APP_ALLOWED_USERS=123456789
APP_POCKETBASE_URL=http://pocketbase_server:8090
```

### Шаг 5: Сохраните файл

```
Ctrl+O (сохранить)
Enter (подтвердить)
Ctrl+X (выйти)
```

### Шаг 6: Перезапустите приложение

```bash
# Вернитесь в главную директорию
cd ~/umbrel

# Перезапустите приложение
./scripts/app restart vault-telegram-vault
```

## Способ 3: Через docker-compose (продвинутый)

### Шаг 1: Подключитесь по SSH

```bash
ssh umbrel@umbrel.local
```

### Шаг 2: Перейдите в директорию приложения

```bash
cd ~/umbrel/app-data/vault-telegram-vault
```

### Шаг 3: Отредактируйте docker-compose.yml

```bash
nano docker-compose.yml
```

### Шаг 4: Добавьте переменные напрямую

Найдите секцию `environment:` и измените на:

```yaml
environment:
  TG_TOKEN: "123456789:ABCdefGHIjklMNOpqrsTUVwxyz"
  PB_URL: "http://pocketbase_server:8090"
  PB_ADMIN: "admin@vault.local"
  PB_PASSWORD: "YourPassword123"
  MASTER_PASSWORD: "YourLongMasterPassword123456"
  ALLOWED_USERS: "123456789"
  WEB_PORT: "8080"
```

### Шаг 5: Сохраните и перезапустите

```bash
# Сохраните файл (Ctrl+O, Enter, Ctrl+X)

# Перезапустите
cd ~/umbrel
./scripts/app restart vault-telegram-vault
```

## Получение необходимых значений

### APP_TG_TOKEN

1. Откройте Telegram
2. Найдите @BotFather
3. Отправьте: `/newbot`
4. Следуйте инструкциям
5. Скопируйте токен

**Подробно:** [ПОЛУЧЕНИЕ-ТОКЕНА-БОТА.md](ПОЛУЧЕНИЕ-ТОКЕНА-БОТА.md)

### APP_ALLOWED_USERS

1. Откройте Telegram
2. Найдите @userinfobot
3. Отправьте любое сообщение
4. Скопируйте ваш ID (только цифры)

### APP_PB_PASSWORD

Придумайте пароль:
- Минимум 8 символов
- Используйте буквы, цифры, символы
- Пример: `SecurePass123`

### APP_MASTER_PASSWORD

Придумайте длинный пароль:
- Минимум 32 символа (рекомендуется)
- Очень важно: если потеряете - данные потеряны навсегда!
- Пример: `MyVeryLongAndSecureMasterPassword123456`

## Проверка после настройки

### Шаг 1: Проверьте логи

```
Umbrel → Apps → Telegram Vault → Logs
```

Должно быть:
```
✅ Starting Telegram Vault bot...
✅ Web UI started on port 8080
✅ Successfully authenticated with PocketBase
✅ Bot started successfully!
```

### Шаг 2: Откройте дашборд

```
Umbrel → Apps → Telegram Vault → Open
```

Должен открыться веб-интерфейс с мониторингом

### Шаг 3: Проверьте бота

1. Откройте Telegram
2. Найдите вашего бота
3. Отправьте: `test.com user pass123`
4. Должен ответить: "saved"

## Troubleshooting

### Переменные не сохраняются

**Решение:** Используйте Способ 2 (через SSH и .env файл)

### Приложение не перезапускается

**Решение:**
```bash
cd ~/umbrel
./scripts/app stop vault-telegram-vault
./scripts/app start vault-telegram-vault
```

### Логи показывают "Missing required environment variables"

**Проверьте:**
1. Все переменные добавлены
2. Нет опечаток в именах переменных
3. Значения не пустые
4. Приложение перезапущено после изменений

### "Failed to authenticate with PocketBase"

**Проверьте:**
1. PocketBase установлен и запущен
2. APP_PB_ADMIN и APP_PB_PASSWORD правильные
3. APP_POCKETBASE_URL правильный

## Пример полной конфигурации

```env
# Обязательные переменные
APP_TG_TOKEN=987654321:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw
APP_PB_ADMIN=admin@vault.local
APP_PB_PASSWORD=SecurePassword123
APP_MASTER_PASSWORD=MyVeryLongAndSecureMasterPassword123456789
APP_ALLOWED_USERS=123456789

# Опциональные переменные
APP_POCKETBASE_URL=http://pocketbase_server:8090
```

## Почему Umbrel не показал форму?

Возможные причины:
1. Версия Umbrel не поддерживает UI для переменных
2. Приложение установлено из Community App Store (ограниченный UI)
3. Нужно настраивать после установки

**Это нормально!** Многие приложения в Umbrel настраиваются после установки.

## Следующие шаги

После успешной настройки:
1. ✅ Проверьте логи
2. ✅ Откройте дашборд
3. ✅ Протестируйте бота
4. ✅ Сохраните MASTER_PASSWORD в надежном месте
