# CI/CD Flow - Автоматическая публикация

## Схема процесса

```
┌─────────────────────────────────────────────────────────────────┐
│                         Разработчик                              │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             │ git push
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                          GitHub                                  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │              GitHub Actions Workflow                      │  │
│  │                                                            │  │
│  │  1. Checkout code                                         │  │
│  │  2. Setup Docker Buildx                                   │  │
│  │  3. Login to Docker Hub                                   │  │
│  │  4. Extract version from umbrel-app.yml                   │  │
│  │  5. Build image (amd64 + arm64)                           │  │
│  │  6. Push to Docker Hub                                    │  │
│  └──────────────────────────────────────────────────────────┘  │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             │ docker push
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                        Docker Hub                                │
│                                                                   │
│  Repository: pitcentr/telegram-vault                             │
│  Tags:                                                            │
│    - latest                                                       │
│    - 1.0.2                                                        │
│                                                                   │
│  Platforms:                                                       │
│    - linux/amd64                                                  │
│    - linux/arm64                                                  │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             │ docker pull
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                          Umbrel                                  │
│                                                                   │
│  1. User installs app from Community App Store                   │
│  2. Umbrel pulls image from Docker Hub                           │
│  3. Starts container with environment variables                  │
│  4. App is ready to use                                          │
└─────────────────────────────────────────────────────────────────┘
```

## Преимущества автоматизации

### ✅ Для разработчика
- Не нужно устанавливать Docker локально
- Не нужно помнить команды сборки
- Не нужно вручную публиковать образы
- Просто `git push` - и всё готово

### ✅ Для пользователей
- Всегда актуальная версия
- Поддержка разных архитектур (x86, ARM)
- Быстрая установка
- Надежность (образы собираются в чистом окружении)

### ✅ Для проекта
- Версионирование образов
- История всех сборок
- Автоматическое тестирование (можно добавить)
- Прозрачность процесса

## Триггеры сборки

Workflow запускается автоматически при:

1. **Push в main/master**
   ```bash
   git push origin main
   ```

2. **Изменении файлов:**
   - `vault-telegram-vault/docker/**` (любые файлы в docker/)
   - `vault-telegram-vault/umbrel-app.yml` (изменение версии)
   - `.github/workflows/docker-publish.yml` (изменение workflow)

3. **Ручной запуск:**
   - GitHub → Actions → Build and Push → Run workflow

## Что происходит при сборке?

### Шаг 1: Подготовка (30 сек)
- Клонирование репозитория
- Настройка Docker Buildx
- Авторизация в Docker Hub

### Шаг 2: Извлечение версии (5 сек)
```bash
VERSION=$(grep '^version:' vault-telegram-vault/umbrel-app.yml | sed 's/version: "\(.*\)"/\1/')
# Результат: 1.0.2
```

### Шаг 3: Сборка образа (2-4 мин)
- Сборка для linux/amd64
- Сборка для linux/arm64
- Использование кеша для ускорения

### Шаг 4: Публикация (30 сек)
- Push образа с тегом `latest`
- Push образа с тегом версии (например, `1.0.2`)

### Общее время: 3-5 минут

## Версионирование

### Автоматическое
При каждой сборке создаются два тега:

1. **latest** - всегда указывает на последнюю версию
   ```yaml
   image: pitcentr/telegram-vault:latest
   ```

2. **Версия** - конкретная версия из umbrel-app.yml
   ```yaml
   image: pitcentr/telegram-vault:1.0.2
   ```

### Обновление версии

1. Измените версию в `umbrel-app.yml`:
   ```yaml
   version: "1.0.3"
   ```

2. Закоммитьте:
   ```bash
   git add vault-telegram-vault/umbrel-app.yml
   git commit -m "Bump version to 1.0.3"
   git push
   ```

3. GitHub Actions автоматически:
   - Соберет новый образ
   - Опубликует с тегами `latest` и `1.0.3`
   - Старые версии останутся доступны

## Мониторинг сборок

### GitHub Actions
1. Откройте репозиторий на GitHub
2. Вкладка "Actions"
3. Список всех сборок с статусами:
   - ✅ Успешно
   - ❌ Ошибка
   - 🔄 В процессе

### Docker Hub
1. https://hub.docker.com/r/pitcentr/telegram-vault
2. Вкладка "Tags" - список всех версий
3. Вкладка "Builds" - история сборок (если настроен Automated Build)

## Безопасность

### Секреты
- `DOCKER_USERNAME` - хранится в GitHub Secrets
- `DOCKER_PASSWORD` - хранится в GitHub Secrets
- Никогда не попадают в логи
- Доступны только в защищенном окружении

### Токены Docker Hub
- Используйте Access Token, а не пароль
- Ограничьте права: только Read, Write, Delete
- Можно отозвать в любой момент
- Не дает доступ к аккаунту

## Troubleshooting

### Сборка не запускается
**Причина:** Изменения не в отслеживаемых файлах
**Решение:** Запустите вручную через GitHub UI

### Ошибка авторизации
**Причина:** Неверные DOCKER_USERNAME или DOCKER_PASSWORD
**Решение:** Проверьте секреты в GitHub Settings

### Образ не появляется на Docker Hub
**Причина:** Репозиторий не создан или приватный
**Решение:** Создайте публичный репозиторий

### Долгая сборка
**Причина:** Первая сборка или изменения в зависимостях
**Решение:** Следующие сборки будут быстрее (кеш)

## Расширенные возможности

### Добавить тесты
```yaml
- name: Run tests
  run: |
    cd vault-telegram-vault/docker
    npm install
    npm test
```

### Добавить линтер
```yaml
- name: Lint Dockerfile
  uses: hadolint/hadolint-action@v3.1.0
  with:
    dockerfile: vault-telegram-vault/docker/Dockerfile
```

### Уведомления
```yaml
- name: Notify on success
  if: success()
  run: echo "Build successful!"
```

## Сравнение подходов

| Критерий | Ручная сборка | GitHub Actions |
|----------|---------------|----------------|
| Время настройки | 5 мин | 10 мин |
| Нужен Docker локально | ✅ Да | ❌ Нет |
| Автоматизация | ❌ Нет | ✅ Да |
| Multi-platform | Сложно | ✅ Легко |
| История сборок | ❌ Нет | ✅ Да |
| Версионирование | Вручную | ✅ Авто |
| Кеширование | Локально | ✅ Облако |

## Рекомендация

**Используйте GitHub Actions** если:
- Планируете частые обновления
- Хотите автоматизацию
- Нужна поддержка ARM64
- Работаете в команде

**Используйте ручную сборку** если:
- Разовая публикация
- Нет доступа к GitHub Actions
- Предпочитаете полный контроль
